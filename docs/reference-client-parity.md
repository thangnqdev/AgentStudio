# Reference client parity audit

This document records evidence-backed parity against the reconstructed Claude Code 2.1.88 source at upstream commit `a8a678cb6244e6770e1e421767ff0987a1d95549`. The reference is research material, not a dependency and not a source-copy target. AgentStudio reimplements observable contracts behind its own Electron, provider, permission, persistence, and renderer boundaries.

Statuses:

- **Implemented**: the reference-facing contract and its important runtime invariant exist.
- **Compatible**: the capability exists, but the public name, protocol, or edge behavior differs.
- **Partial**: a useful subset exists while a named runtime boundary is still missing.
- **Missing**: no equivalent model-facing capability exists.
- **Not applicable**: experimental/internal output plumbing that is not a normal client capability.

## Runtime and client subsystems

| Subsystem | Status | Highest-value gap |
|---|---|---|
| Streaming model loop, retries, fallback models | Implemented | OpenAI-compatible SSE framing is bounded across arbitrary chunks/final frames; string/array/cumulative content, fragmented/index-less tool calls, standard cached prompt counts, and split cache-read/cache-creation usage are normalized and traced. Provider-native prompt-cache controls and non-Chat-Completions event grammars are not fully equivalent. |
| Context budgeting and compaction | Partial | Automatic local projection/compaction, bounded visual-token estimates, multimodal tool continuity and durable deferred-schema continuity exist. `/compact` now performs a provider-free, main-owned deterministic reduction: renderer IPC strips attachment bytes/capabilities, main validates unique IDs under 1,000-message/12-million-character limits, only an actually smaller result is accepted, and Pre/Post audit brackets success. The renderer replaces history only against the unchanged thread/message-array snapshot, using a local-derived `agent` summary plus recent originals rather than a renderer-authored system message. The optional note is retained text, not provider re-summarization. Composer files use picker-derived, fingerprint-pinned, expiring capabilities; raw renderer paths are rejected. Renderer projection plus strict main validation also strip path/token/preview/media data from bounded chat-history persistence, and main uses the active workspace rather than a renderer-selected history key. Expired files require reattachment, and reference provider-driven attachment/delta compaction remains non-equivalent. |

| Durable sessions/checkpoints | Implemented | Agent transcripts/checkpoints are durable. Next-step/next-turn model context and open-renderer toasts have independent one-shot receipts. |

Elicitation matches only `questions` and excludes question/answer content; `TeammateIdle` matches the persisted member name after protocol delivery; `CwdChanged` covers worktree enter/exit, not one-off command cwd. File-change hooks match workspace-relative paths and are composed independently with LSP notification; shell/MCP/external mutations are not inferred. Child compaction sends only an allow-listed event enum over strict RPC; the parent owns hook lookup and audit identity. `/hooks` lists only normalized effective ID/event/matcher/action-type metadata; context/reason/label values stay out of the renderer. Instruction/permission/notification/elicitation/subagent/teammate/worktree/cwd/config/file/compaction/stop/end hooks are audit-only; CLI-only `Setup`, executable command/HTTP/prompt hooks and several plugin component types remain unsupported until separately sandboxed. |

Production model/conversation loops run in filtered child processes; catalog, tool execution, approval, hooks, audit, checkpoints and trace persistence remain authoritative in the parent through strictly validated RPC. |

A keyboard-navigable slash-command palette with real navigation/thread actions also exists. `/model` and `/permissions` open persisted keyboard pickers with current-value focus; `/resume`/`/continue` searches up to 100 current-workspace paused/failed checkpoints by normalized title/error/task ID; `/rename` persists an explicit current-session title without later auto-title overwrite; `/context` shows estimated used/remaining context in a bounded grid; `/status` summarizes local version/provider/model/permission/workspace/Git state but does not probe account/API health; `/hooks` inspects sanitized effective configuration; `/compact` offers optional preservation text and applies only a main-validated, stale-guarded local reduction. ## Ordered parity roadmap

1. **Remaining process compatibility**: sandboxed native PowerShell without weakening the proven filesystem and process boundaries.
2. **Client/provider edges**: remaining reference command-specific modal breadth beyond model/permissions/resume/rename/context/status/hooks/compact, cross-project session browsing, IDE diff-tab control/onboarding, native provider PDF document blocks, provider-native prompt-cache controls, and non-Chat-Completions event semantics.
3. **Optional OS scheduling integration**: only if a separately reviewed installer/service design can preserve explicit authorization and uninstall cleanup.

The full parity objective is therefore **not complete**. Passing tests for one milestone proves that milestone only; this matrix must be updated from source evidence as each remaining boundary is implemented and verified.
